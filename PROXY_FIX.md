# 代理问题解决方案

## 问题描述
在运行 `npm run dev` 时遇到 `TypeError: fetch failed` 错误，原因是系统中设置了代理环境变量（`http_proxy` 和 `https_proxy`），导致 Cloudflare Miniflare 无法正常启动。

## 根本原因
- 系统中存在全局代理环境变量（通常来自 Clash 等代理软件）
- Wrangler/Miniflare 检测到代理后会尝试通过代理访问本地服务
- 代理服务器无法正确处理这些内部请求，导致连接失败

## 解决方案
创建了一个 PowerShell 启动脚本 `dev-no-proxy.ps1`，在启动开发服务器前清除所有代理相关的环境变量。

### 如何使用

现在使用以下命令启动开发服务器（替代原来的 `npm run dev`）：

```bash
npm run dev:clean
```

这个命令会：
1. 清除当前会话中的所有代理环境变量
2. 验证清除结果
3. 启动 Vite 开发服务器

### 文件说明

- **dev-no-proxy.ps1**: PowerShell 启动脚本
- **package.json**: 添加了 `dev:clean` npm 脚本

### 注意事项

- 如果你需要在开发时访问外网（比如使用 Ollama API），可能需要根据具体情况调整这个脚本
- 这个脚本只影响当前终端会话的环境变量，不会修改系统级别的设置
- 如果关闭代理软件后依然有问题，可能需要检查系统环境变量中是否永久设置了代理

## 长期解决方案（可选）

如果你经常需要在有代理的环境下开发，建议在系统环境变量中添加：
- 变量名: `NO_PROXY`
- 变量值: `localhost,127.0.0.1,::1`

这样可以让大部分工具在访问本地服务时自动绕过代理。
